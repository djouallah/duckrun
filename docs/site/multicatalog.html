Part 1 · attach three catalogs of different kinds to one session

A writable OneLake lakehouse (the primary), a read-only Fabric Warehouse, and a local scratch dir — bound as catalogs, addressed as catalog.schema.table.

1 attach: a read-only Fabric Warehouse + a writable local scratch catalog

# the warehouse is read-only (managed by the SQL engine — not writable via OneLake):
conn.attach(warehouse_path, name="warehouse", storage_options={"bearer_token": tok},
            read_only=True)
# a local scratch dir, writable (inherits the session default):
conn.attach(scratch_path, name="scratch")

Part 2 · write across catalogs, then JOIN and write the mart back to the lakehouse

Seed the writable lakehouse and the local scratch catalog, JOIN across them in one conn.sql, and write the result BACK to the aemo lakehouse as a new Delta table.

4 seed the aemo lakehouse: 'multicatalog_demo.regional_demand' (5 regions × 24 hours)

CREATE OR REPLACE TABLE multicatalog_demo.regional_demand AS
SELECT r.region, h.hour,
       round(500 + 300 * sin((h.hour + ascii(r.region)) / 3.0) + 40 * ascii(r.region), 1) AS demand_mw
FROM (VALUES ('A'), ('B'), ('C'), ('D'), ('E')) r(region)
CROSS JOIN range(0, 24) h(hour)

6 cross-catalog JOIN → write back: 'multicatalog_demo.mart_region_demand' in the aemo lakehouse

# one query spans the writable lakehouse (regional_demand) and the local scratch
# catalog (region_lookup); the result is written back to the lakehouse by name:
mart = conn.sql("""
    SELECT k.state, k.region_name,
           round(sum(d.demand_mw), 1) AS total_mw,
           round(avg(d.demand_mw), 1) AS avg_mw
    FROM multicatalog_demo.regional_demand d
    JOIN scratch.dbo.region_lookup k ON k.region = d.region
    GROUP BY k.state, k.region_name""")
mart.write.mode("overwrite").saveAsTable("multicatalog_demo.mart_region_demand")
